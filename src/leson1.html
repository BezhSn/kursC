Язык C# и .NET

На сегодняшний день язык C# является одним из наиболее популярных языков программирования в мире. Согласно Tiobe Index, C# уверенно занимает четвертую строчку рейтинга после таких популярных языков как Java, Python и C.

На C# пишется огромное количество приложений — от мелких утилит для Windows до крупных web-порталов, а современные возможности платформы .NET позволяют создавать кроссплатформенные приложения практически под любую операционную систему, включая такие как Linux, Mac OS и так далее.

Язык C# и .NET

Нередко, когда речь заходит о языке программирования C# имеется в виду технология .NET и наоборот — при упоминании .NET имеют в виду C#. Однако, эти понятия хоть и являются близкими, но не являются синонимами.

Чтобы было понятно о чем идет речь — посмотрим на то, что из себя в настоящее врем представляет собой платформа .NET.

Платформа .NET

.NET — это платформа для разработки любых приложений: для Интернета, мобильных устройств, настольных ПК, игр, Интернета вещей и так далее.

Можно выделить следующие преимущества платформы .NET:

  1. Поддержка нескольких языков программирования. В основе всей платформы .NET лежит CLR (Common Language Runtime) — общеязыковая среды выполнения. Поэтому, кроме C# приложения для .NET могут быть написаны, например, на таких языках как F#, Visual Basic, Visual C++ и даже на ныне «мёртвом» диалекте — Delphi.NET.
  2. Кроссплатформенность. Платформа .NET в современной её реализации позволяет разрабатывать приложения для Windows, Linux, Mac OS, Android и других операционных систем.
  3. Мощная система классов. Какое бы приложение на каком бы языке мы не разрабатывали, в любом случае мы задействуем систему классов .NET, а так называемые упаковка-преобразование и распаковка-преобразование позволяют работать с любым типом данных (например, со строками или целочисленными значениями) как с объектами, что значительно упрощает процесс изучения возможностей системы классов .NET.
  4. Разнообразие технологий. CLR и система классов .NET являются основой целого стека технологий, применяемых в .NET. Так, например, для разработки приложений с графическим интерфейсом мы можем использовать технологии WPF, Windows Forms или Universal Windows Platform (UWP). С одной стороны, обилие всевозможных различных технологий может отпугивать новичков в .NET, но, с другой стороны — позволяет выбрать для решения конкретной задачи именно ту технологию, которая подойдет наиболее точно.
  5. Автоматическая сборка мусора. Последнее в списке, но далеко не последнее по значимости преимущество. CLR самостоятельно удаляет не используемые объекты из памяти, благодаря чему, нам больше нет необходимости заботится о своевременном уничтожении неиспользуемых объектов, а значит — утечки памяти сведены к минимуму (надо очень постараться, чтобы сделать утечку памяти, например, при работе в C#).

.NET Framework, Mono и .NET Core

Существуют различные реализации платформы .NET основными из которых являются следующие:

  • .NET Framework — это реализация .NET для разработки приложений исключительно для Microsoft Windows. Эта платформа активно развивалась с 2002 по 2019 год. На данные момент, последней версией является версия 4.8.
  • Mono — ещё одна реализация .NET, но уже предназначенная в первую очередь для разработки небольших по размеру приложений, например для операционной системы Android, iOS или tvOS.
  • .NET Core — относительно новая реализация .NET предназначенная для кроссплатформенной разработки.

На данный момент, наиболее перспективной реализацией .NET можно считать .NET Core которая предназначена в том числе и для создания высокопроизводительных систем и приложений.

Среда выполнения (CLR)

Среда выполнения — это, по сути, виртуальная машина на которой выполняется программа . Операционная система является частью среды выполнения, но не входит в среду выполнения .NET.
В зависимости от выбранной реализации .NET используются различные среды выполнения, а именно:

  • Среда CLR для .NET Framework
  • Среда CoreCLR для .NET Core
  • .NET Native для универсальной платформы Windows
  • Среда выполнения Mono для Xamarin.iOS, Xamarin.Android, Xamarin.Mac и платформы Mono для рабочего стола.

Как работают программы в .NET

Компиляция и запуск приложений происходит следующим образом:

  1. Код из любого языка преобразовывается в код, написанный на общем языке (Common intermediate language или CIL). Этот язык можно условно назвать ассемблером для .NET.
  2. полученный IL-код передаётся среде выполнения (CLR), которая берёт необходимые методы и функции из соответствующей реализации .NET, например из .NET Core.
  3. Конечный результат передаётся на процессор и выполняется программа.
На первый взгляд выглядит немного нерационально с точки зрения производительности и кажется, что любая программа, написанная на C# будет уступать по скорости работы той же программе, написанной, например, на Delphi. Однако, на деле всё обстоит не так и приложения разработанные в C# работают ничуть не медленнее, чем написанные на Delphi или C++.

Дело в том, что при выполнении приложения происходит JIT-компиляция (Just-In-Time) в машинный код, который затем выполняется. При этом, в конкретный момент времени будет компилироваться лишь та часть приложения, к которой непосредственно идет обращение. Если же мы обратимся к другой части кода, то она будет скомпилирована из CIL в машинный код, а уже скомпилированная часть приложения сохраняется до завершения работы программы. В конечном итоге JIT-компиляция позволяет серьезно повысить производительность приложений.
